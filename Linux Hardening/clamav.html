ClamAV Antivirus

Open-source malware scanner — install, update, scan, automate

Debian / Ubuntu Open Source CLI-Driven Daemon Mode

What does ClamAV actually do?

ClamAV is an open-source antivirus engine maintained by Cisco Talos. It scans files, directories, and mail streams against a signature database of known malware, trojans, and malicious scripts. On Linux systems it's particularly useful for scanning files that pass through your machine destined for Windows users, auditing uploaded content, and satisfying compliance requirements that mandate antivirus coverage.

It doesn't run in the background watching files in real time by default — clamscan is an on-demand scanner. The daemon variant (clamd) is available for integration with mail servers and web applications where speed matters. For most personal and small-server use cases, scheduled on-demand scans are sufficient.

STEP 01

Verify Installation

Check whether ClamAV is already present on your system by querying the binary directly:

terminal
clamscan --version

If a version string is returned (e.g. ClamAV 1.0.x), skip to Step 03. "Command not found" means proceed to Step 02.

STEP 02

Install ClamAV

Install both the scanner and the automatic signature updater from the standard Debian/Ubuntu repositories:

terminal
sudo apt update && sudo apt install clamav clamav-freshclam -y

This installs two components:

clamav
The clamscan scanner binary and its signature database engine
clamav-freshclam
The freshclam daemon that keeps virus definitions up to date automatically
STEP 03

Update Virus Definitions

ClamAV is only as effective as its definitions. Before scanning anything, ensure the database is current. The freshclam service normally handles this automatically, but you should run a manual update first — especially on a fresh install:

terminal
# Stop the freshclam service before running manually — they conflict sudo systemctl stop clamav-freshclam sudo freshclam sudo systemctl start clamav-freshclam

You'll see freshclam connect to the ClamAV mirror network and download the latest main.cvd, daily.cvd, and bytecode.cvd definition files. This can take a minute on the first run.

The freshclam service checks for updates multiple times per day once running. You only need to run it manually when you want to force an immediate update.

STEP 04

Run Your First Scan

Scan a specific directory recursively. The flags below give you a clean, actionable output:

terminal
# Scan your home directory — recursive, show infected files only clamscan -r --bell -i ~/ # Scan the entire system (slow — use for thorough audits) sudo clamscan -r --bell -i /

Flag breakdown:

-r
Recursive — scan all subdirectories
--bell
Audible alert if a threat is found
-i
Only print infected files — suppresses the noise of clean results
sudo
Required for scanning system directories outside your home

A full system scan can take 20–60 minutes depending on disk size. Use ~/ or a specific path for faster targeted scans.

STEP 05

Reading the Results

When the scan completes, ClamAV prints a summary. Here's what each line means:

CLEAN
No threats found. The summary will show Infected files: 0. With the -i flag, you'll see no file output — that's expected and means everything is clean.
INFECTED
A threat signature matched. The infected file path and signature name are printed — e.g. /home/user/file.sh: Eicar-Signature FOUND. Investigate before deleting; false positives do occur, particularly with scripts and compressed archives.
ERRORS
Files that couldn't be read. Usually permission errors on system files — not a security concern. Running with sudo eliminates most of these.
SUMMARY
Scanned directories, scanned files, infected files, time elapsed. The line you care about is Infected files: N. Anything above 0 warrants investigation.
STEP 06

Quarantine or Remove Infected Files

If threats are found, you have two options. Quarantine is safer — it moves the file rather than destroying it, so you can review it before committing to deletion:

terminal
# Move infected files to a quarantine directory instead of deleting sudo mkdir -p /quarantine sudo clamscan -r -i --move=/quarantine ~/ # Or remove immediately (irreversible — use with caution) sudo clamscan -r -i --remove ~/

Before removing, search the ClamAV signature name online — e.g. at VirusTotal or the ClamAV malware portal — to confirm it's a genuine detection and not a false positive.

Scripts, package installers, and pentest tools are common false positive sources. When in doubt, quarantine first.

STEP 07

Automate Weekly Scans with Cron

Set up a scheduled scan so your home directory is checked automatically every week. The results are written to a log file you can review at any time:

terminal
# Open your crontab for editing crontab -e

Add this line to run every Sunday at 02:00, logging output to /var/log/clamav-scan.log:

crontab entry
0 2 * * 0 clamscan -r -i --log=/var/log/clamav-scan.log ~/

To review the most recent scan log:

terminal
cat /var/log/clamav-scan.log

The freshclam service handles definition updates independently — your definitions stay current automatically regardless of when you scan.
