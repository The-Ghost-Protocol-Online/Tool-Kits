Comprehensive Installation and Architectural Overview for Debian Systems

00 Core Concepts

Wazuh is an enterprise-grade, open-source security platform designed for threat detection, integrity monitoring, incident response, and compliance. It integrates SIEM (Security Information and Event Management) with XDR (Extended Detection and Response) capabilities into a unified stack.

Wazuh Manager
The central processing unit. It analyzes data from agents, triggers alerts based on a multi-thousand rule set, and manages remote configuration.
Wazuh Indexer
A highly scalable, full-text search and analytics engine. It indexes and stores security alerts generated by the manager.
Wazuh Dashboard
The visual interface for data visualization and platform management, providing out-of-the-box dashboards for PCI DSS, GDPR, and MITRE ATT&CK.

01 Manual Repository Provisioning

Standard automation scripts often fail on Debian Testing (Trixie) due to missing management dependencies like software-properties-common. Manually importing the GPG key and repository ensures package availability.

Shell Command
curl -s https://packages.wazuh.com/key/GPG-KEY-WAZUH | gpg --dearmor | sudo tee /usr/share/keyrings/wazuh.gpg > /dev/null echo "deb [signed-by=/usr/share/keyrings/wazuh.gpg] https://packages.wazuh.com/4.x/apt/ stable main" | sudo tee /etc/apt/sources.list.d/wazuh.list sudo apt update

02 Firewall Configuration

Ensure the following network ingress rules are applied to UFW to facilitate Dashboard access and secure Agent-to-Manager communication.

UFW Policy
sudo ufw allow 443/tcp sudo ufw allow 1514/tcp sudo ufw allow 1515/tcp

03 Deployment with System Override

To execute the installation on Debian Trixie, the -i (ignore system check) flag must be utilized. This bypasses the platform verification logic that otherwise halts the script on non-LTS releases.

All-In-One Assistant
sudo ./wazuh-install.sh -a -i
Note: Hardware requirements include a minimum of 4GB RAM and 2 CPU cores to support the Indexer's JVM heap requirements.

04 Administrative Access

Upon installation completion, the system generates unique credentials for the admin account. If necessary, these can be recovered from the secure local log.

Credential Recovery
sudo cat /var/ossec/etc/wazuh-passwords.txt
